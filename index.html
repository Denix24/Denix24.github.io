Capstone Project — HVALIDATE

Presenteeism Clinical Validation Dashboard

HVALIDATE is a clinical validation dashboard designed to help evaluate productivity loss caused by presenteeism while protecting employee privacy. The system focuses on cohort-level outputs only (no individual drill-down) and supports stakeholders in understanding trends, severity staging, and estimated economic impact.

My Role
  • Dashboard development and visual design choices
  • Analytics workflow logic (validation, scoring, aggregation)
  • Security and privacy-by-design documentation and controls
Key Capabilities
  • Upload synthetic survey CSV aligned to required schema
  • Compute validation rate and highlight inconsistencies
  • Severity staging metrics and cohort summaries
  • Economic impact estimates at an aggregated level
Privacy-by-Design Summary
We use synthetic data for demonstrations; outputs are aggregated at cohort level; small groups are suppressed using k-anonymity thresholds; and access control planning includes IAM-based authorization for online deployment.
Python Streamlit Data Validation Cohort Analytics HIPAA-aligned Principles GCP Deployment (planned)

Metrics (What we track)

  • Validation Rate: % of records passing rule checks
  • Inconsistency Flags: mismatches or out-of-range values
  • Severity Staging: distribution across low/medium/high stages
  • Cohort Summaries: group-level counts, proportions, trends
  • Economic Impact: estimated cost ranges at cohort level

Deployment

Currently runs locally on laptops (Python + Streamlit). Online deployment targets a controlled environment with IAM access, HTTPS, and centralized logging.
